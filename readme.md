# 📄 Документооборот (DocFlow)  
## Курсовая работа

---

# 1. Введение

Данный проект представляет собой backend-систему документооборота, реализованную на базе FastAPI с использованием асинхронной работы через SQLAlchemy (async) и базы данных PostgreSQL.

Основная цель системы — автоматизация работы с документами внутри организации.

Система позволяет:

- создавать документы
- назначать исполнителей
- контролировать сроки выполнения
- отслеживать просрочки
- вести историю изменений
- общаться внутри документа (чат)
- получать уведомления
- управлять доступами пользователей (RBAC)

---

## Основные задачи системы

- управление жизненным циклом документов  
- контроль исполнения задач  
- централизованное хранение информации  
- прозрачность действий пользователей  

---

## Архитектурные принципы

Система построена с учетом:

- расширяемости (можно добавлять новые функции)
- разделения ответственности (слои системы)
- поддержки асинхронной нагрузки
- интеграции с внешними системами (например Bitrix)
